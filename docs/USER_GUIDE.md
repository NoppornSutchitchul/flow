## ภาพรวมระบบ

Flow ช่วยให้:

1. **หน้าฟร้อนท์ / หัวหน้า** สร้างคำขอจากห้องพักหรือพื้นที่โรงแรม (ของใช้, บริการ, ช่างซ่อม)
2. **ระบบ** มอบหมายงานให้พนักงานที่ออนไลน์และว่าง (ตามแผนก / โซนงาน)
3. **แม่บ้าน / ช่าง** รับงานจาก **คิวงาน (My Queue)** แล้วอัปเดตสถานะจนส่งมอบ
4. **ผู้จัดการ** ดูแดชบอร์ด KPI, สต็อกต่ำ, รายงาน และเกณฑ์เวลาแจ้งเตือน

การอัปเดตแบบเรียลไทม์ใช้ **WebSocket** — รายการคำขอและสถิติจะ refresh โดยไม่ต้อง reload หน้าบ่อย

| ไอคอน | หน้า | คำอธิบายสั้นๆ |
|-------|------|----------------|
| บ้าน | ภาพรวม (Dashboard) | KPI งานค้าง, เกินเวลา, สต็อกต่ำ, ตารางงานด่วน |

| คลิปบอร์ด | คำขอ (Requests) | รายการคำขอทั้งหมด กรองตามวัน/สถานะ |

## 1) ภาพรวม (Dashboard) — หัวหน้า / ฟร้อนท์ / ผู้จัดการ

**เส้นทาง:** `/` (ไอคอนบ้าน)

### สิ่งที่แสดง

- **การ์ด KPI** เช่น งานค้าง, เกินกำหนด, สต็อกต่ำ, Rush
- **การแจ้งเตือน** เช่น DND รอฟร้อนท์โทรแขก, ไม่มีพนักงานออนไลน์
- **ตารางคำขอที่กำลังดำเนินการ** — แถวกระพริบสีตามเวลา (เหลือง → แดง → เกินกำหนด)

### วิธีใช้

1. คลิกการ์ด KPI เพื่อไปหน้ารายการที่เกี่ยวข้อง (ถ้ามีลิงก์)
2. คลิกแถวในตารางเพื่อเปิด **รายละเอียดคำขอ**
3. ใช้ร่วมกับปุ่ม **+** เพื่อสร้างงานใหม่

---

## 2) แจ้งคำขอ (Quick Request) — ปุ่ม + กลาง dock

**เปิดจาก:** ปุ่ม **+** ด้านล่าง หรือ `/?quick=1`

### ขั้นตอนสร้างคำขอ

1. **เลือกห้อง / สถานที่**
   - พิมพ์เลขห้อง (เช่น `2506`) หรือเลือกจากรายการ
   - ระบบแสดงประเภทห้อง / วิว / ขนาด (ถ้ามีในฐานข้อมูล)

2. **เลือกบริการหรือของใช้**
   - ค้นหาในช่อง "เลือกบริการหรือของใช้..."
   - เลือกรายการ — แยกแผนกอัตโนมัติ (แม่บ้าน / ช่าง / ฟร้อนท์ / bell)
   - ปรับจำนวน (ของใช้) หรือใส่หมายเหตุต่อรายการ

3. **เวลาส่งมอบ**
   - **ส่งทันที** — ระบบมอบหมายพนักงานที่ออนไลน์ทันที
   - **นัดล่วงหน้า** — เลือก delay / วันเวลา / รายวัน (งานจะค้างจนถึงเวลา)

4. **มอบหมาย (ถ้ามีตัวเลือก)**
   - เลือกแม่บ้าน / ช่างเฉพาะคน หรือปล่อย **อัตโนมัติ**

5. **วิธีส่งมอบ** (ปุ่มด้านล่างขวา)
   - กดกริ่งและส่งมอบ / วางหน้าประตู / รับที่ฟร้อนท์ (เลือกจากเมนูลูกศร)

6. กดปุ่มส่ง (ตามวิธีส่งมอบที่เลือก)

### ข้อความแดง "ไม่มีพนักงานออนไลน์"

แสดงเมื่อ **ไม่มีพนักงานแผนกนั้นออนไลน์และรับงานได้** ในขณะนั้น

**วิธีโชว์ให้ผ่าน:**

- เปิดแท็บ/เครื่องอื่น login เป็น **ช่าง** (`Anan` / `Somsak`) หรือ **แม่บ้าน** แล้วปล่อยหน้าคิวงานค้างไว้ (ให้ WebSocket ส่งสถานะ online)
- จากนั้นกลับมาสร้างคำขอแผนกนั้นอีกครั้ง

### สต็อกไม่พอ

ถ้าของใช้ในคลังไม่พอ รายการจะขึ้น **หมดสต็อก** — ลบรายการหรือลดจำนวนก่อนส่ง

| การกระทำ | ใครทำได้โดยทั่วไป |

## 5) คิวงาน (My Queue) — แม่บ้าน / ช่าง

**เส้นทาง:** `/queue`

### โซนในหน้า

| โซน | ความหมาย |
|-----|----------|
| งานว่าง / Available | งานที่มอบหมายแล้ว รอรับ |
| กำลังทำ | เริ่มงานแล้ว |
| พักไว้ | Pause ชั่วคราว |
| DND | แขกติดป้ายห้ามรบกวน — รอฟร้อนท์ |
| สำเร็จวันนี้ | ส่งมอบแล้วในวันนี้ |

### ขั้นตอนทำงานมาตรฐาน

1. **รับงาน** — จากการ์ดงานที่มอบหมายให้
2. **เริ่มงาน** — เมื่อเดินทางถึงห้อง / เริ่มซ่อม
3. (ถ้าจำเป็น) **พัก** หรือ **แจ้ง DND**
4. **ส่งมอบ** — เมื่อเสร็จ (กริ่ง / วางหน้าประตู ตามที่คำขอกำหนด)

### โซนงาน (Work zone)

แม่บ้านชั้นอาจตั้ง **ตึก · ชั้น** ในหน้าคิว — ระบบใช้จับคู่งานใกล้เคียง

### นับถอยหลังเวลา

การ์ดแสดงเวลาที่เหลือตาม **เกณฑ์การตอบสนอง** ที่ admin ตั้งไว้ (ดูหัวข้อ Settings)

## 10) สถานะคำขอ (อ้างอิง)

```
pending → assigned → in_progress ⇄ paused → delivered
                              ↘ dnd (รอฟร้อนท์)
                              ↘ cancelled
```

- **pending** — รอมอบหมายหรือรอพนักงานออนไลน์
- **assigned** — มีคนรับผิดชอบแล้ว ยังไม่เริ่ม
- **in_progress** — กำลังทำ
- **paused** — พักชั่วคราว
- **dnd** — Do Not Disturb
- **delivered** — ส่งมอบแล้ว
- **cancelled** — ยกเลิก

## 11) การทำงานเบื้องหลัง (สำหรับโชว์ / นำเสนอ)

เมื่ออธิบายให้ผู้ชม demo:

1. **Frontend (React)** — UI, ภาษา, dock navigation
2. **Backend (FastAPI)** — business rules, มอบหมายงาน, สต็อก, รายงาน
3. **WebSocket** — อัปเดตรายการคำขอและ presence แบบ real-time
4. **Presence** — พนักงานต้อง **login และเชื่อมต่อ WS** ถึงจะนับว่าออนไลน์รับงานได้
5. **Auto-assign** — เลือกคนว่าง + ใกล้โซนห้อง (แม่บ้าน) หรือ workload ต่ำสุด
6. **Escalation** — งาน assigned แต่ไม่เริ่ม เกินเกณฑ์ → ส่งต่อหัวหน้าแม่บ้าน / ผจก.

แผนภาพสถาปัตยกรรมเพิ่มเติม: [ARCHITECTURE.md](./ARCHITECTURE.md)

## 13) แก้ปัญหาเบื้องต้น

| อาการ | แนวทาง |
|--------|--------|
| Login ไม่ได้ | ตรวจ username/password, ลอง `Admin` / `1234` |
| สร้างคำขอไม่ได้ — ไม่มีพนักงานออนไลน์ | ให้ staff แผนกนั้น login และเปิดหน้าคิวงานค้างไว้ |
| หน้าว่างหลัง login | บทบาทไม่มีสิทธิเมนู — ลองบัญชีอื่น |
| ข้อมูลไม่อัปเดต | รีเฟรชหนึ่งครั้ง; ตรวจว่า WebSocket ไม่ถูก firewall บล็อก |
| บันทึกเกณฑ์เวลาไม่ได้ | ต้อง login เป็น **Admin** |
| เซิร์ฟเวอร์ช้าครั้งแรก | instance อาจ cold start — รอ 10–30 วินาทีแล้วลองใหม่ |
